# Business Value

ForgePulse 的商业价值不是“替代设备工程师”，而是把设备工程师每天要手工拼接的证据链自动整理出来：传感器趋势、报警、SOP、维修记录、现场描述、质量风险和工单动作。

## Target Customer

首选试点客户：

- 新能源电池制造企业。
- 有涂布、辊压、分切、化成、PACK 等连续产线。
- 已有 MES/SCADA/QMS/报警系统，但异常诊断仍依赖资深工程师经验。
- 有停机复盘、质量追溯和新人培养压力。

首个落地部门：

- 设备工程部门。
- 质量工程部门。
- 生产现场管理团队。

## Business Pain

### 1. 故障证据分散

产线异常通常不只来自一个系统：

- 报警系统只告诉你发生了什么报警。
- 传感器趋势告诉你指标如何变化。
- SOP 告诉你标准处理流程。
- 维修记录告诉你过去类似问题怎么解决。
- 质量系统告诉你有没有影响物料。

现场工程师需要在多个系统之间切换，人工拼出因果链。ForgePulse 的价值是把这些线索整合成一份可追溯诊断。

### 2. 排障依赖资深工程师

新能源电池产线设备复杂，很多问题是耦合异常：温度、张力、风机、驱动电流、膜厚互相影响。新人能看到报警，但不一定知道优先排查什么。

ForgePulse 不替代资深工程师，而是把资深工程师的判断路径结构化：

- 哪些事实确定。
- 哪些证据支持根因。
- 哪些动作优先。
- 哪些点需要人工确认。

### 3. 复盘难沉淀

故障处理结束后，复盘报告经常靠人工补写，格式不一致，证据引用不完整。ForgePulse 可以把诊断过程直接转成复盘报告和工单草案，降低知识沉淀成本。

## Value Metrics

这些指标应作为试点目标，不应包装成已验证承诺。

ForgePulse 中的价值卡片只在 `case_manifest.business_context` 显式配置基线后计算，并展示计算假设。没有现场参数时显示“未配置”，避免使用看似精确但无法追溯的通用行业数字。

| Metric | Baseline Problem | ForgePulse Target |
|---|---|---|
| MTTR 初步定位时间 | 工程师人工查多系统，耗时不稳定 | 缩短初步定位时间 30%-50% |
| 重复故障沟通成本 | 相似故障反复问资深工程师 | 复用历史维修记录和 SOP 证据 |
| 异常物料隔离及时性 | 质量风险和设备异常关联滞后 | 自动提示受影响时间窗和质量动作 |
| 复盘报告一致性 | 人工报告格式不统一 | 自动生成结构化报告 |
| 新人工程师上手 | 排障经验难系统传承 | 用证据链和动作板辅助训练 |

## Why It Can Become a Product

ForgePulse 的市场化可行性来自三个点：

1. **只读接入，低风险试点**

   第一阶段只读接入数据，不参与设备控制，不改变现场安全流程。

2. **Evidence-grounded，便于信任**

   每个根因和动作都能追溯到报警、传感器、SOP 或维修记录。

3. **垂直行业聚焦**

   项目不做泛 AI 助手，而是聚焦电池制造产线设备诊断。窄场景更容易证明价值。

## Competitive Differentiation

普通 Agent demo 通常是“输入问题，输出建议”。ForgePulse 的差异是：

- 多源工业数据融合。
- 故障模式库。
- 根因评分拆解。
- 诊断过程追踪。
- 证据图。
- 角色化动作。
- 工单和复盘报告导出。
- 离线确定性运行，LLM 只是增强项。

## Commercial Boundary

ForgePulse 第一阶段适合做：

- 设备异常诊断辅助。
- 停机复盘报告生成。
- 维修知识沉淀。
- 新人工程师排障训练。
- 厂内试点工具。

第一阶段不适合做：

- 自动控制设备。
- 安全联锁替代。
- 未经审核的生产决策自动执行。
- 跨行业泛化平台。
