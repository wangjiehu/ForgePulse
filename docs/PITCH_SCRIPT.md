# Pitch Script

## 30-second Version

ForgePulse 是面向新能源电池产线的设备异常诊断 Agent。它不是聊天机器人，而是工程师工作台：把传感器趋势、报警、SOP、维修记录和现场描述整合成可追溯的根因排序、证据链、角色化处置动作、维修工单和复盘报告。项目支持离线运行，不依赖模型 Token，当前内置 5 个工业案例，71 项自动检查全部通过。

## 5-minute Version

### 1. Problem

新能源电池产线的异常诊断很难，因为异常不是单点问题。

一次涂布线异常可能同时出现：

- 干燥区温度漂移；
- 张力振荡；
- 风机频率变化；
- 驱动电流升高；
- 膜厚漂移；
- 质量风险。

现场工程师需要在报警、趋势图、SOP、维修记录和质量系统之间来回切换。资深工程师可以凭经验判断，但新人很难快速建立完整证据链。

### 2. Solution

ForgePulse 做的事情是把这些分散线索变成一个可追溯诊断闭环。

输入：

- 传感器 CSV；
- 报警日志；
- SOP；
- 维修记录；
- 现场描述。

输出：

- 异常时间线；
- 根因候选排序；
- 每个根因的评分拆解；
- 证据链；
- 角色化处置动作；
- 工单草案；
- 复盘报告；
- 保守的业务价值估算。

### 3. Why It Is Different

普通 Agent demo 往往是聊天框，答案好不好很难验证。

ForgePulse 的差异是 evidence-grounded：

- 每个根因都引用真实 evidence id；
- 每条 evidence 能追溯到报警、传感器、SOP 或维修记录；
- 每个动作都连接到根因或安全要求；
- 自动评测会检查证据、根因、动作和禁止声明。

### 4. Demo Flow

打开工作台后，先展示五个案例：

1. 干燥 2 区温控回路异常 + 张力波动；
2. 风机变频频率异常 + 干燥不均风险；
3. 驱动电流升高 + 辊面污染阻力。
4. 缺少关键温度证据，Agent 必须停止确认并请求补数；
5. 报警与风机传感器证据冲突，Agent 必须请求现场核验。

选择第一个案例：

- 看 data quality；
- 看 diagnostic process；
- 看 timeline；
- 看 root cause ranking；
- 展开 evidence chain；
- 查看 role-based actions；
- 导出 Markdown report。

然后切换证据不足和证据冲突案例，证明系统不仅会给答案，也知道何时不能确认。

### 5. Business Value

ForgePulse 的价值不是替代工程师，而是辅助工程师更快完成初步定位和复盘。

试点目标：

- 缩短初步排障时间 30%-50%；
- 提高复盘报告一致性；
- 帮助新人学习资深工程师的排查路径；
- 更快识别受影响物料时间窗；
- 把历史维修经验沉淀为可检索知识。

### 6. Safety Boundary

ForgePulse 只做诊断辅助和报告生成。

它不控制设备，不绕过安全联锁，不替代工程师确认，不做自动质量放行。

### 7. Closing

ForgePulse 已经具备一个试点级工业 Agent 的核心形态：多案例、可评测、可追溯、可离线运行、能生成报告。下一步可以在真实工厂做只读数据试点，对历史故障进行回放验证。
